# 7 Percent — Mobile App Setup

## Overview

This is the React Native (Expo) mobile app for the 7 Percent fitness platform. It shares the same backend (`server/`) as the web app — no backend changes are needed beyond CORS (already updated).

The single `mobile/` codebase compiles to both platforms:
- `mobile/android/` — Android native build (generated by `expo prebuild`)
- `mobile/ios/` — iOS native build (generated by `expo prebuild`)

---

## Prerequisites

- Node.js 18+
- Expo CLI: `npm install -g expo-cli eas-cli`
- For iOS builds: macOS with Xcode 15+
- For Android builds: Android Studio with SDK 34+

---

## Quick Start (Development)

### 1. Install dependencies
```bash
cd mobile
npm install
```

### 2. Configure the server URL

Open `src/config/server.js` and set `DEV_API_URL`:

| Where you're testing       | Value                            |
|---------------------------|----------------------------------|
| iOS Simulator              | `http://localhost:3001`          |
| Android Emulator           | `http://10.0.2.2:3001`           |
| Physical device (any OS)   | `http://192.168.X.X:3001` (your computer's LAN IP) |

### 3. Start the backend
```bash
# From the repo root:
cd server && npm run dev
```

### 4. Start the Expo dev server
```bash
cd mobile
npm start        # opens Expo CLI menu
npm run ios      # opens iOS Simulator directly
npm run android  # opens Android Emulator directly
```

Then scan the QR code with the **Expo Go** app on your phone, or press `i` / `a` in the terminal.

## Generate Native Build Folders (android/ and ios/)

Run this once to create the native project directories:

```bash
cd mobile
npx expo prebuild
```

This generates:
- `mobile/android/` — the Android Gradle project
- `mobile/ios/` — the Xcode project

After `prebuild`, you can open:
- `mobile/android/` in Android Studio
- `mobile/ios/7Percent.xcworkspace` in Xcode

### Platform-specific prebuild:
```bash
npx expo prebuild --platform android
npx expo prebuild --platform ios
```

---

## Production Builds (EAS Build)

### Set up EAS
```bash
eas login
eas build:configure
```

Update `app.json` → `extra.eas.projectId` with your EAS project ID.

### Build for both platforms:
```bash
npm run build:all
```

### Build individually:
```bash
npm run build:android   # .apk / .aab
npm run build:ios       # .ipa
```

## Architecture

```
mobile/
├── App.jsx                     Root: QueryClient + AuthProvider + Navigation
├── src/
│   ├── api/client.js           API client (mirrors web, uses AsyncStorage)
│   ├── config/server.js        Server URL config (update for your environment)
│   ├── lib/
│   │   ├── AuthContext.jsx     Auth state (mirrors web AuthContext)
│   │   ├── subscriptionUtils.js Reused unchanged from web
│   │   └── theme.js            Shared colour tokens
│   ├── navigation/index.jsx    React Navigation (Tab + Stack)
│   ├── screens/
│   │   ├── auth/               Login · Register · ResetPassword
│   │   └── main/               Home · Workouts · Nutrition · Progress · Profile
│   │                           Coaching · Challenges · Leaderboard
│   │                           Badges · Friends · Subscription
│   └── components/
│       ├── ui/                 Button · Input · Card
│       └── workout/WorkoutCard.jsx
└── assets/                     App icons + splash screen
```

## What's Reused from the Web App

| Web                              | Mobile                          | Status      |
|----------------------------------|---------------------------------|-------------|
| `server/` (entire backend)       | Unchanged                       | ✅ 100% reused |
| `src/api/client.js`              | `src/api/client.js`             | ✅ Adapted (AsyncStorage) |
| `src/lib/AuthContext.jsx`        | `src/lib/AuthContext.jsx`       | ✅ Adapted (no window.location) |
| `src/lib/subscriptionUtils.js`   | `src/lib/subscriptionUtils.js`  | ✅ Identical copy |
| All server functions             | Called via same API             | ✅ 100% reused |
| All Mongoose models              | Served via same API             | ✅ 100% reused |
| React Query patterns             | Same queries / mutations        | ✅ Same pattern |

## Differences from the Web App

| Web                     | Mobile                              |
|------------------------|-------------------------------------|
| Radix UI / shadcn/ui   | React Native StyleSheet             |
| TailwindCSS            | Shared `theme.js` colour tokens     |
| react-router-dom       | React Navigation (Stack + Tab)      |
| localStorage           | AsyncStorage                        |
| Recharts               | Simplified stats (charts TBD)       |
| Stripe.js              | Deep-link to billing portal         |
